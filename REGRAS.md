# Regras de Comportamento para Agentes

## Padrão de Nomenclatura (PT-BR)
A partir de agora, **todos os novos arquivos, variáveis, funções e componentes** criados neste sistema (tanto no frontend quanto no backend) devem ser nomeados utilizando o padrão **Português do Brasil (PT-BR)**, a menos que a linguagem ou framework exija um termo específico em inglês.

Exemplos:
- Em vez de `DEPLOY_MULTIPLE_CLINICS.md`, usar `IMPLANTACAO_MULTIPLAS_CLINICAS.md`.
- Em vez de `getAppointments()`, usar `obterAgendamentos()`.
- Em vez de `is_active`, usar `ativo` ou `esta_ativo`.
